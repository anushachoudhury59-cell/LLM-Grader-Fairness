Does a customer's name move the grader's score?

Same 40 support replies, only the customer's name changed (White-coded vs. Black-coded), graded by two different judge models. Section 1 compares the name-swap delta against the judge's own repeat-to-repeat noise floor and against what a real quality difference moves the score by. Section 2 checks whether any single name is worse than the group average suggests.

Section 1 - Aggregate: swap delta vs. noise vs. quality gap

One run (run_1.csv), graded 3 times per provider with the cache cleared between each repeat, so the noise floor reflects real repeat-to-repeat wobble, not a cached zero.

Section 2 - Per-name: is any single name worse, hidden inside the group average?

Aggregating White vs. Black can hide a single outlier name if others in the same group happen to pull the average back up. Each name's effect below is computed against its own ticket's baseline score (averaged across every run that ticket appeared in), not the raw group mean - so a name that happens to draw harder tickets by chance isn't penalized for it. Bars whose 95% interval doesn't cross zero are flagged - but with 10 names tested per provider at 95% confidence and no multiple-comparisons correction, roughly 0.5 false flags are expected from chance alone, so treat a single flagged name (especially one whose interval barely excludes zero, or with no other same-group name pointing the same direction) as a lead to re-test with more bootstrap runs, not a confirmed finding.
